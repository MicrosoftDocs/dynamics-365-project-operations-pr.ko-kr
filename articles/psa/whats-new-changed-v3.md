# <a name="whats-new-or-changed-in-project-service-automation-version-3"></a>Project Service Automation 버전 3의 새로운 내용 또는 변경 내용
[!INCLUDE[cc-applies-to-psa-app-1x-2x](../includes/cc-applies-to-psa-app-1x-2x.md)]

[!include [rename-banner](~/includes/cc-data-platform-banner.md)]

이 항목은 버전 2 또는 버전 1과 버전 3 사이의 Project Service Automation의 UI(사용자 인터페이스), 기능 및 용어에 대한 변경 사항 정보를 제공합니다.

## <a name="project-scheduling"></a>프로젝트 일정
이전 버전에서 WBS(작업 분할 구조)로 알려진 프로젝트 일정은 일정으로 이름이 바뀌었으며 **일정** 탭을 클릭하여 액세스합니다. 

![프로젝트 일정](media/psa-schedule-01.png)

일정은 이제 현대적이고 접근 가능한 상호 작용을 위한 새로운 표면을 갖습니다. 그러나 기본 Project Service Automation 일정 엔진은 변경되지 않았습니다. 일정 표의 리본에 있는 컨트롤 단추를 사용하면 이전 버전의 Project Service Automation과 유사한 일정과 상호 작용할 수 있습니다. 일정에 대한 추가 변경 사항은 다음과 같습니다.

- **Gantt 차트** - Gantt 차트가 더 이상 존재하지 않습니다. 새로운 Gannt 시각화는 향후 업데이트에서 돌아옵니다.
- **열 머리글** - 열 제목 옆의 아래 표시기를 클릭하여 표에서 열 헤더를 숨길 수 있습니다. 
- **열** - **열 추가** 를 클릭하여 숨겨진 열을 표시할 수 있습니다. 
- **거래 범주** - **거래 범주** 조회가 일정 표에 추가되었으며 기본적으로 표시됩니다. 
 
## <a name="project-templates"></a>프로젝트 템플릿
프로젝트 템플릿 기능에 대해 다음과 같은 변경 사항이 적용되었습니다.

### <a name="create-a-project-template"></a>프로젝트 템플릿 만들기 
이전 버전의 Project Service Automation과 유사한 버전 3에서 프로젝트 템플릿을 만들 수 있습니다. 템플릿에는 일정만 포함될 수 있으며 일정에 할당을 포함할 수 있지만 필수는 아닙니다. 일정에 할당이 있는 경우 일반 리소스에 대해서만 할당할 수 있습니다. 일반 리소스에 대한 리소스 요구 사항을 생성할 수 있지만 템플릿에서 실제 리소스로 예약할 수는 없습니다. 템플릿에서 팀에 실제 리소스를 예약할 수 없습니다. 

### <a name="create-a-template-from-an-existing-template"></a>기존 템플릿에서 새 템플릿 만들기
Project Service Automation 버전 3의 기존 템플릿에서 새 프로젝트 템플릿을 만들면 다음과 같은 일이 발생합니다. 

- 원본 프로젝트의 일정이 템플릿에 복사됩니다. 
- 일반 리소스가 팀에 복사되고 일반 리소스 할당이 복사됩니다. 일반 리소스에 대한 요구 사항은 복사되지 않습니다. 

### <a name="create-a-template-from-an-existing-project"></a>기존 프로젝트에서 새 템플릿 만들기
기존 프로젝트에서 새 프로젝트 템플릿을 만들면 다음과 같은 일이 발생합니다. 

- 원본 프로젝트의 일정이 템플릿에 복사됩니다. 
- 일반 리소스가 팀에 복사되고 일반 리소스 할당이 보존됩니다. 일반 리소스에 대한 요구 사항은 복사되지 않습니다.    
- 할당되거나 할당되지 않은 명명된 리소스는 팀에서 제거되고 일반 리소스로 대체됩니다.
- 고객 정보가 있는 경우 제거됩니다. 
- 견적 및 계약에 대한 참조가 있는 경우 제거됩니다. 

### <a name="create-a-project-from-a-template"></a>템플릿에서 프로젝트 만들기
Project Service Automation 버전 3에서, 기존 템플릿으로 새 프로젝트 템플릿을 만들면 다음과 같은 일이 발생합니다.

- 일정, 팀 및 할당이 새 프로젝트에 복사됩니다.   
- 시작 날짜는 사용자가 선택한 복사 날짜 또는 날짜입니다.   
- 템플릿에 리소스 요구 사항이 있는 일반 팀 구성원의 경우 요구 사항이 자동으로 복사되거나 생성되지 않습니다. 생성해야 합니다. 

## <a name="copy-a-project"></a>프로젝트 복사
Project Service Automation 버전 3에서, 프로젝트를 복사하면 다음과 같은 일이 발생합니다. 

- 예상 시작 날짜는 복사되지만 변경할 수 있습니다.  
- 프로젝트 일정 및 작업이 복사됩니다. 
- 일반 리소스 및 해당 할당이 복사됩니다. 일반 리소스에 대한 리소스 요구 사항은 복사되지 않습니다. 다시 생성해야 합니다. 
- 실제 리소스 및 해당 할당이 복사되지 않습니다. 대신 일반 리소스로 대체됩니다. 
- 실제 결과는 새 프로젝트에 복사되지 않습니다. 

## <a name="move-a-scheduled-project"></a>예약된 프로젝트 이동
기존 프로젝트의 일정을 앞으로 이동하면 다음과 같은 일이 발생합니다. 

- 작업 날짜가 이동 기간에 맞게 자동으로 이동됩니다. 
- 할당된 일반 리소스는 할당된 상태로 유지됩니다.   
- 프로젝트가 이동되기 전에 생성되는 경우, 일반 리소스에 대한 요구 사항은 동일하게 유지되며 자동으로 다시 생성되지 않습니다. 작업 이동으로 인해 새 할당을 반영하도록 다시 생성해야 합니다. 
- 실제 리소스에 대한 할당은 작업 날짜 이동에 맞게 변경됩니다. 실제 리소스에 대한 예약은 변경되지 않습니다. 조정 보기를 사용하여 예약을 수정해야 합니다. 
- 예약이 있지만 할당이 없는 팀 리소스는 변경되지 않습니다. 
- 실제는 이동하지 않습니다. 

## <a name="estimates"></a>추정
예상은 **리소스 할당** 및 **예상** 두 탭으로 분할되었습니다. **리소스 할당** 탭에는 작업량 예상이 포함되어 있으며 시간 단계별 보기에서 작업에 대한 리소스 할당이 표시됩니다. 일정 엔진이 생성한 내용에 따라 예상을 편집할 수 있습니다.

![작업에 대한 작업량 예측 및 리소스 할당을 표시하는 리소스 할당 탭](media/resource-assignments-tab-02.png)

**예상** 탭에는 리소스 할당에 대한 비용 및 영업 금액이 표시됩니다. 금액은 읽기 전용입니다. 이제 비용 및 영업 가격 책정은 일정에 따라 팀 구성원 할당에서 발생합니다. 즉, 할당없는 작업이 있는 경우 작업이 할당되지 않은 버킷 아래에 표시됩니다. 즉, 기본 가격책정 차원인 **역할** 이 없으면 프로젝트와 연결된 고객 또는 계약/견적이 있는 경우 예상 비용이나 영업이 없습니다. 

![예상 탭에 비용 및 영업 금액 표시](media/estimates-tab-03.png)
  
범주는 일정 보기의 작업에서도 지원됩니다. 예상의 시간 단계별 보기에서 범주별로 그룹화하면 특히 프로젝트에 경비 예상이 있는 경우 더 나은 환경을 제공할 수 있습니다. 경비 예상은 별도의 탭에서 표를 사용하여 입력됩니다. 

경비 예상은 **경비 예상** 탭의 표에서 입력됩니다. 

![경비 예상 표를 보여주는 경비 예상 탭](media/expense-estimates-tab-04.png)

## <a name="resource-management"></a>리소스 관리
Project Service Automation 버전 3에서는 새로운 통합 클라이언트 UI와 예약과 할당 간의 관계가 변경되면서 일반 또는 실제 리소스가 있는 프로젝트의 인력 배치가 버전 2및 버전 1에서 크게 변경되었습니다. 그러나 **실제** 리소스와 **일반** 모두 예약 가능한 리소스의 개념은 팀 구성원, 요구 사항, 할당 및 예약과 동일하게 유지됩니다.   

![리소스 선택기 사용](media/resource-management-05.png)

### <a name="assign-a-real-bookable-resource"></a>실제 예약 가능한 리소스 할당 
Project Service Automation 버전 3에서는 예약 및 작업 할당이 이전 버전의 Project Service Automation처럼 밀접하게 얽혀 있지 않습니다. 팀 표를 사용하여 시장에서와 유사한 **실제** 팀 구성원을 예약할 수 있습니다.

일정에 따라 리소스 선택기를 사용하여 팀 보기에서 만든 팀 구성원을 선택한 다음 작업에 할당할 수 있습니다. 예약을 지나서도 작업을 계속 할당할 수 있습니다. **조정** 탭을 사용하여 예약 및 할당에 차이가 있는 팀 구성원을 조정합니다.

리소스 선택기는 프로젝트의 팀 구성원을 표시합니다. 또한 일정 탭에서 리소스 선택기를 사용하여 아직 프로젝트 팀에 속하지 않은 다른 예약 가능한 리소스를 검색하고 선택할 수 있습니다. 작업에 할당할 수 있으며 프로젝트 팀의 일부가 됩니다. **일정 게시판** 또는 **조정** 탭을 사용하여 예약해야 합니다.

### <a name="assign-a-generic-bookable-resource-on-a-task-and-project-team-and-then-fulfill-with-a-real-resource-via-schedule-board"></a>작업 및 프로젝트 팀에 일반 예약 가능한 리소스를 할당한 다음 일정 게시판을 통해 실제 리소스로 이행 
Project Service Automation 버전 3에서는 일반 리소스에 팀 생성 기능이 사용되지 않습니다. 대신 일정의 리소스 셀에 일반 리소스의 위치 이름을 입력하여 일정에서 일반 리소스를 만들고 직접 할당할 수 있습니다. 또는 셀의 리소스 아이콘을 선택하여 리소스 선택기를 사용하여 만들려는 일반 리소스의 이름을 입력합니다. 이렇게 하면 일반 리소스 팀 구성원의 역할 및 조직 구성 단위를 설정할 수 있는 빠른 만들기 패널이 열립니다. 리소스를 만든 후 작업에 할당되며 해당 일반 리소스를 일정의 다른 작업에 계속 할당할 수 있습니다.    
 
모든 적절한 작업에 리소스를 할당한 경우 리소스 요구 사항을 생성한 다음 **일정 게시판** 에서 직접 예약하거나 리소스 요청을 제출하여 리소스 요구 사항을 충족할 수 있습니다. 일반 리소스를 팀 구성원 표에 직접 추가할 수도 있습니다. 

일반 리소스는 리소스 요구 사항이 없고 리소스 요구 사항이 생성될 때까지 프로젝트의 시작/종료 날짜와 함께 프로젝트 팀에 추가됩니다. 요구 사항을 생성하려면 일반 리소스를 선택하고 **생성** 을 클릭합니다. 이제 요구 사항 링크가 표시되고 필요한 시간이 할당된 시간으로 채워집니다. 링크를 클릭하여 요구 사항을 열고 업데이트할 수 있습니다.
  
예약이 완료되고 명명된 리소스에 의해 완전히 수행되면 일반 리소스가 명명된 리소스로 대체되고 일정에 대한 할당이 명명된 리소스로 업데이트됩니다. 

이제 요구 사항에 대한 제안된 리소스가 별도의 섹션 대신 탭에 저장됩니다.

### <a name="multiple-named-resources-fulfilling-a-generic-resource"></a>여러 명명된 리소스로 일반 리소스 이행
요구 사항이 여러 리소스로 충족되면 일반 리소스가 팀에 남아 작업에 할당됩니다. 예약된 명명된 팀 구성원은 직책의 일부로 할당되지 않습니다. 프로젝트 관리자는 실제 리소스에 필요한 작업을 할당할 수 있습니다.  **조정** 보기는 여러 리소스에 걸쳐 여러 작업 할당에 대한 예약을 나눌 수 있게 해줍니다. 요건을 구성하는 작업 번들이 있는 경우와 같이 더 복잡한 시나리오에서는 프로젝트 관리자가 배정하려는 방식의 의도를 시스템에서 가정해야 하기 때문에 이 작업은 자동으로 수행되지 않습니다. 시스템이 의도를 이해할 수 없기 때문에 가정이 의도한 것과 다를 수 있으며 올바르지 않거나 예측할 수 없는 결과가 발생할 가능성이 있습니다. 프로젝트 관리자가 **조정** 보기를 사용하여 리소스를 할당할 때까지 예측 가능한 결과는 일반 리소스가 배정된 상태로 유지된다는 것입니다.

### <a name="reconciliation"></a>조정
**조정** 탭은 각 프로젝트 팀 구성원의 예약 및 모든 할당을 표시합니다. 셀에 수 개월~수 일의 시간을 나타낼 수 있는 시간을 보여줍니다. 이 보기로 프로젝트 관리자가 프로젝트 팀을 위한 팀원 예약과 배정을 조정하는 데 도움이 됩니다. 이는 예약 및 작업 할당이 밀접하게 결합되지 않아 프로젝트를 계획할 때 더 많은 유연성을 허용하기 때문에 유용합니다. 

![조정 탭은 프로젝트 팀 구성원의 예약 및 할당 표시](media/resource-reconciliation-tab-06.png)

각 리소스에 대해 보기는 팀 구성원의 예약과 작업 할당 롤업 간의 차이를 취하고 프로젝트의 예약 및 할당에 발생할 수 있는 다음 두 가지 차이점을 보여 주었습니다. 

- **예약 부족** – 리소스에 예약보다 많은 과제가 있을 때 예약 부족이 발생합니다. 이 능력은 예약되지 않았므로 프로젝트 관리자는 부족을 충당하기 위해 리소스 예약을 연장하여 수정할 수 있습니다. 
- **초과 예약** – 리소스가 프로젝트에 예약되었지만 과업에 할당되지 않은 경우 초과 예약이 발생합니다.  예컨대, 과업 할당이 발생하기 전에 리소스를 예약한 경우 수용할 수 있는 항목이 될 수 있습니다. 그러나 다른 경우에는 리소스가 할당되지 않을 수 있으며 PM은 다른 프로젝트에 생산 능력을 사용할 수 있도록 리소스 예약을 취소하는 것을 고려해야 합니다. 

예약(예약 부족)이 없는 리소스에 대한 작업 할당이 있는 경우, 집계 예약 부족을 선택한 다음 **예약 연장** 을 선택할 수 있습니다. 여기에서 리소스 부족 및 가용성을 해결하는 데 필요한 예약을 볼 수 있습니다. 
 
## <a name="time-and-expense"></a>시간 및 경비
이 섹션에서는 Project Service Automation 버전 3의 시간, 경비 및 승인의 변경에 대한 정보를 제공합니다. Dynamics 365 Project Service Automation 솔루션의 일부로 통합 인터페이스 프레임워크를 활용하기 위해 **시간 항목** 기능이 새로 고쳐졌습니다. 이를 통해 반응형 디자인을 따르는 일관되고 균일한 사용자 인터페이스를 제공하여 모든 화면 크기 또는 장치에서 최적의 보기를 제공합니다. 

### <a name="landing-page"></a>랜딩 페이지
확장 불가능한 사용자 지정 시간 항목 환경은 버전 3에서 더 이상 사용되지 않습니다. 대신 확장 가능하고 접근 가능한 네이티브 표 환경이 있습니다. 왼쪽의 사이트맵을 사용하여 시간 항목 기능에 액세스할 수 있습니다. 이 변경으로, 더 이상 한 번에 일주일 동안 시간을 입력할 수 없습니다. 대신 표에서 매일 시간 항목을 만들어야 합니다. 몇 시간 항목을 만든 후 사용자는 이 주제의 후반부에 설명된 **복사** 함수를 사용하여 시간 항목을 대량으로 만들 수 있습니다. 

![시간 항목 방문 페이지](media/time-entry-landing-page-07.png)
 
### <a name="create-new-time-entries"></a>새 시간 항목 만들기 
리본에서 **신규** 를 클릭하여 분, 시간 또는 일의 기간을 입력하는 시간 항목을 빠르게 만다는 페이지를 엽니다. 이렇게 하려면 수량과 함께 h, m 또는 d를 입력하기만 하면 됩니다.  

![시간 항목 빨리 만들기](media/quick-create-time-entry-08.png)

조회 필드는 시스템 보기에서 지원됩니다. 예를 들어 프로젝트 정보를 입력한 후 **프로젝트 작업** 필드는 기본적으로 **내 열린 프로젝트 작업** 보기로 설정됩니다. 사용자에게 할당되지 않은 작업에 대한 시간 항목을 만들려면 조회에서 **보기 변경** 을 클릭한 다음 **모든 활성 프로젝트 작업** 을 선택합니다. 시간 항목이 만들어지고 표에 표시되면 표에서 직접 모든 라인 값을 편집할 수 있습니다.  

### <a name="bulk-createcopy"></a>대량 만들기/복사 
약간의 시간 항목을 만든 후 대량으로 복사 기능을 사용하여 추가 시간 항목을 대량으로 만들 수 있습니다. **복사** 를 클릭하여 **복사** 대화 상자를 엽니다. **시작 기간: 시작 날짜** 에서, 복사해야 하는 시간 범위에서 날짜 범위를 설정합니다. **끝 시간: 시작 날짜** 에서, 만들어야 하는 시간 항목의 날짜를 지정합니다. **복사** 를 클릭하여 **끝 기간** 에 표시된 해당 요일과 관련된 시간 항목을 복사합니다. 예를 들어, 지난 주 월요일의 시간 항목은 **끝 기간** 으로 표시된 주의 월요일로 복사됩니다. 

![대량으로 시간 항목 복사](media/bulk-copy-time-entry-09.png)
 
### <a name="import-data"></a>데이터 가져오기 
할당 및 교환은 사용자가 예약을 가져와야 할 때의 날짜 범위를 지정할 수 있도록 동일한 UI 패턴을 따릅니다. 그런 다음 **초안** 시간 항목으로 복사해야 하는 예약을 명시적으로 선택해야 합니다. 버전 3에서는 표 및 캘린더에서 **추천** 시간 항목의 패턴을 더 이상 볼 수 없습니다.  

### <a name="change-in-calendar-control"></a>캘린더 컨트롤의 변경
버전 3에서는 사용자 지정 일정 컨트롤에서 벗어나 UC 캘린더를 사용하여 한 주의 시간 항목을 표시하고 있습니다. 이 캘린더를 사용하면 일, 주 및 월을 볼 수 있습니다. 

> [!NOTE]
> 캘린더의 제한 사항은 이 컨트롤이 개별 캘린더 항목에 대한 작업을 지원하지 않는다는 것입니다. 예를 들어 하나 이상의 캘린더 항목을 선택하고 해당 항목을 제출하거나 삭제할 수 없습니다. 캘린더 항목을 클릭하면 추가 작업에 대한 **시간 항목 엔터티** 페이지가 열립니다. 

### <a name="extensibility"></a>확장성
**시간 및 경비 항목 엔터티에서만 사용자 지정 필드의 날짜 캡처** - 시간 항목은 편집 가능한 표, 읽기 전용 표 및 플랫폼의 캘린더 컨트롤을 사용합니다. 이러한 모든 컨트롤은 기본 컨트롤이므로 사용자 지정을 지원합니다. Project Service Automation 버전 3에서는 추가 사용자 지정 필드를 추가하고 조회 필드를 설정하고 사용자 지정 보기로 백업할 수 있습니다. 사용자 지정 필드의 선택된 값을 기반으로 사용자 지정 비즈니스 논리를 설정할 수도 있습니다.  

**시간 및 경비 항목에서 사용자 지정 필드의 데이터 캡처 그리고 제출 및 승인 흐름을 지원하는 엔터티를 통한 전파** - 시간 항목의 일반적인 처리는 다음 다이어그램에 표시됩니다.

![시간 항목 흐름 프로세스](media/process-time-entries-10.png)

비즈니스 요구 사항에 따라 시간 및 경비 엔터티가 사용자 지정 가격 책정 크기를 캡처하고 이전 그래픽의 모든 엔터티를 통해 사용자 지정 가격 책정 크기의 시간 및 항목 리소스에 의해 설정된 값을 전파해야 한다고 규정한 경우 [가격 책정 크기로 사용자 지정 필드 설정](set-up-pricing-dimensions.md)

시간 및 경비 엔터티가 사용자 지정 비가격 측정 기준을 캡처하고 값을 전파해야 하는 비즈니스 요구 사항을 지원하려면 가격 책정 크기 설정을 사용하고 경비 또는 청구 요금 없이 가격 측정치로 사용자 지정 측정 기준을 표현할 수 있습니다. 또 다른 시나리오는 모든 엔터티에서 동일한 필드 이름을 사용하여 각 엔터티에 사용자 지정 필드를 추가하는 것입니다. 사용자 지정 플러그 인을 만들어 거래 확보 경로 및 거래 연결 엔터티를 사용하여 제출/승인 흐름에 참여하는 엔터티의 레코드를 연결할 수 있습니다.  

### <a name="delegate-time-and-expense-entry"></a>위임 시간 및 경비 항목
Common Data Service 플랫폼은 한 사용자가 다른 사용자를 가장하는 것을 지원하지 않으므로 Project Service Automation 버전 3에서는 위임된 시간 및 경비 항목에 대한 지원이 없습니다. 그러나 파트너와 고객은 버전 3에서 위임된 시간 항목 환경을 지원할 수 있도록 해결 방법을 활용했습니다. 이는 해결 방법일 뿐 완전한 해결책이 아니므로 제한 사항이 허용되는 경우에만 이 방법을 사용하는 것이 중요합니다. 

> [!IMPORTANT]
> 이 정보는 파트너/고객이 사용자 지정 구현을 위해 제안된 지침으로만 간주해야 합니다. 제품 팀은 지원 채널을 통해 이 기능에 대한 공식적인 지원을 제공하지 않습니다.

### <a name="customization-details"></a>사용자 지정 세부 정보 
사용자 지정을 사용하면 환경 만들기 및 편집에 **예약 가능한 리소스** 를 추가할 수 있으므로 사용자가 **예약 리소스** 필드를 기록되어야 하는 시간 및 경비 항목이 필요한 다른 사용자로 변경하여 대리인 역할을 할 수 있습니다. 다음 단계는 시간 항목 위임을 다룹니다. 경비 항목 위임에도 동일한 정보가 적용됩니다. 
 
1.  위임된 사용자가 프로젝트 및 프로젝트 작업에 대한 전역 보안 액세스 권한을 가지고 있는지 확인합니다. 
1.  **시간 항목** 엔터티의 필드인 **예약 가능한 리소스** 는 **빠른 만들기** 페이지에 노출되지 않으므로 추가해야 합니다.

    -또는-

    리소스에 대해 작성된 시간 항목만 보려면 **예약 가능한 리소스** 열이 포함된 사용자 지정 보기를 만듭니다. 이 보기에 대한 앱 모듈 디자이너에 사용자 지정을 게시하여 **시간 항목** 페이지에서 **보기 선택기** 아래에 표시됩니다. 프로젝트 이외의 시간 항목에 대한 관리자 설정을 처리하는 두 개의 플러그 인이 있습니다.

    - PreValidateTimeEntryCreate
    - PreValidateTimeEntryUpdate
 
1. **관리자** 필드를 **예약 가능한 리소스** 필드에 할당된 사용자의 관리자에게 덮어쓰는 새 플러그 인을 만듭니다. 대역외(OOB) 플러그 인(사전 유효성 검사)과 동일한 **실행 단계** 를 사용하고 OOB 플러그 인(1보다 큰)보다 높은 **실행 순서** 를 사용합니다. 이렇게 하면 OOB 플러그 인 이후에 사용자 지정 플러그 인이 실행됩니다.  
 
### <a name="end-user-experience"></a>최종 사용자 환경
1.  빨리 만들기 페이지에서 시간 항목을 만들 때 프로젝트 및 프로젝트 작업 세부 정보를 입력한 다음 시간 항목을 기록해야 하는 **예약 가능한 리소스** 필드의 사용자를 선택합니다. 
2.  기본적으로 이 필드는 로그인한 사용자에게 기본값이지만 사용자가 이 필드를 오버로드한 경우 선택한 **예약 가능한 리소스에** 대해 시간 항목이 만들어집니다.
3.  이러한 레코드에 대해 만든 시간 항목을 제출하면 프로젝트의 승인자를 위해 항목이 예상대로 큐에 대기됩니다. 
4.  다른 사용자에 대해 생성된 시간 항목을 기억하면 시간 항목이 다른 사용자로 설정된 **예약 가능한 리소스** 필드를 사용하여 **초안** 상태로 반환됩니다. 
5.  선택적으로 사용자 지정 보기로 전환하여 다른 사용자에 대해 생성된 시간 항목을 필터링할 수 있습니다. 
 
### <a name="limitations"></a>제한 사항
**복사** 및  **가져오기** 기능은 로그인한 사용자의 컨텍스트에서만 작동합니다. 즉, 예약 가능한 리소스로 로그인한 사용자를 위해 만든 시간 항목을 복사하거나 가져올 수 없습니다.

위의 **사용자 지정 세부 정보** 섹션에서 4단계가 완료된 경우에만 프로젝트에 포함되지 않은 시간 항목은 예약 가능한 리소스 관리자에게 승인을 위해 라우팅됩니다. 그렇지 않으면 다른 사용자에 대한 프로젝트 시간 항목이 로그인한 사용자의 관리자에게 잘못 라우팅됩니다. 

### <a name="other-changes"></a>기타 변경 사항 
**예약 및 작업** 기능이 제거되었습니다. 

## <a name="multidimensional-pricing"></a>다차원 가격 책정
유연성을 극대화하고 다양한 비즈니스 요구 사항을 충족하기 위해 Project Service Automation의 버전 3은 가격 책정 차원 세트를 비용 및 청구 요금에 대한 개별 응용 프로그램을 지원합니다. 차원 값을 기본값으로 설정한 다음 리소스 프로파일링에서 프로젝트 실제까지 시간 항목에 이르기까지 원가 계산 및 가격 책정 프로세스에 전파할 수 있습니다. 고객별 구성 및 수정 또는 확장은 표준 사용자 지정 가능 인프라를 활용합니다.

Project Service Automation에는 기본 가격 차원 및 역할 및 리소스 단위 집합이 제공되며 각 역할 및 조직 구성 단위 조합에 대한 가격 및 비용을 설정할 수 있습니다.

버전 3의 가격 책정 차원으로 이러한 즉시 사용 필드를 계속 사용하려는 Project Service Automation 고객의 경우 눈에 띄는 변경 사항이 없습니다. 평소와 같이 Project Service Automation을 계속 사용할 수 있습니다. 그러나 다른 추가 특성을 사용하여 리소스에 대한 가격 또는 비용이 필요한 경우 버전 3을 사용하면 Project Service Automation에 사용자 지정 가격 책정 차원을 추가할 수 있습니다. 사용자 지정 가격 책정 차원의 확장은 복잡한 구성 환경입니다.

Project Service Automation 버전 3에서는 견적 및 계약에 대한 설정 및 관리 측면이 변경되었습니다. 다음 섹션에서는 자세한 정보를 제공합니다.

### <a name="set-up-chargeability-options"></a>청구 가능성 옵션 설정
버전 1과 2에서는 견적 라인 또는 계약 내용의 맨 위 탐색에 있던 **청구 가능성** 보기를 사용하여 특정 견적 및 계약에 대한 역할 및 범주에 대한 청구 가능성 설정을 수행했습니다. 또한 이러한 역할 및 비용 범주에 대한 가격을 설정할 수 있는 위치이기도 합니다.

버전 3을 통해 역할 및 경비 범주별 청구 가능성 옵션 설정은 견적 또는 계약 내용 수준에서 수행됩니다. 가격 책정 설정은 청구 가능성 설정과는 별개입니다. 상단 탐색을 사용하지 않고도 **견적 라인** 및 **계약 내용** 페이지의 탭으로 **청구 가능성 역할** 및 **청구 가능성 범주** 를 찾을 수 있습니다.

![청구 가능한 역할](media/chargeable-12.png)
 
청구 가능성 역할 및 청구 가능성 범주의 설정은 즉시 편집 가능한 표 컨트롤도 활용합니다. 각 역할 및 범주에 대해 견적 및 계약 단계 동안 청구 유형에 대한 지원되는 옵션은이전 버전에서 **청구 가능성** 및 **비청구 가능성** 으로 변경되지 않습니다. 견적 또는 계약 단계에서는 **무료** 가 지원되지 않습니다. **무료** 는 시간 또는 경비 승인 중에만 지원됩니다.  
 
### <a name="create-and-edit-custom-pricing-for-a-project-service-automation-quote-and-project-contract"></a>Project Service Automation 견적 및 프로젝트 계약에 대한 사용자 지정 가격 책정 작성 및 편집
버전 1과 2에서는 특정 견적 및 계약에 대한 사용자 지정 가격표를 사용하여 **청구 가능성** 보기에서 **가격 편집** 을 사용하여 수행했습니다. **청구 가능성** 보기는 견적 라인 또는 계약 내용의 맨 위 탐색에 있습니다. 또한 역할 및 경비 범주의 청구 가능성 옵션을 설정할 수 있는 위치이기도 합니다.

버전 3을 기준으로 Project Service Automation 견적 및 Project Service Automation 프로젝트 계약에 사용자 지정 프로젝트 가격표를 만들고 사용하는 것은 청구 가능성 설정과 분리되었습니다. Project Service Automation 견적 및 Project Service Automation 프로젝트 계약에는 **프로젝트 가격표** 라는 새 탭이 나타납니다. 이 탭에는 Project Service Automation 견적 또는 프로젝트 계약에 연결된 모든 프로젝트 가격표의 관련 보기가 표시됩니다. 프로젝트 견적 또는 계약에 이미 연결된 기존 가격표에서 사용자 지정 가격표를 만들려면 **사용자 지정 가격 책정 만들기** 를 클릭합니다. 이렇게 하면 관련된 모든 가격표의 복사본이 만들어지고 견적 또는 계약에 첨부됩니다. 이제 가격표를 열고 역할 또는 비용 범주 가격을 수정하여 가격 변경 사항이 이 견적 또는 계약에만 적용되도록 할 수 있습니다.
