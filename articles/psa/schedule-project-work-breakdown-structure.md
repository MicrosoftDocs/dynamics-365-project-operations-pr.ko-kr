# <a name="schedule-a-project-with-a-work-breakdown-structure-project-service"></a>작업 분할 구조로 프로젝트 일정 짜기(Project Service)

[!include [banner](../includes/psa-now-project-operations.md)]

[!INCLUDE[cc-applies-to-psa-app-1x-2x](../includes/cc-applies-to-psa-app-1x-2x.md)]

프로젝트 일정은 수행해야 할 작업, 그 작업을 수행할 리소스, 해당 작업 완료에 필요한 시간을 설명합니다. 프로젝트 일정은 제 때 납품하는 것에 관련된 모든 작업을 반영합니다. 프로젝트 시작 단계에서 처음 수행할 일 중 하나는 바로 프로젝트 일정을 짜는 것입니다. 프로젝트 일정을 수립하려면, 작업 분할 구조를 만들어야 합니다.  
  
 작업 분할 구조로 프로젝트 구조를 만들면 다음과 같은 이점이 있습니다.  
  
- 작업을 관리할 수 있는 업무로 분할  
  
- 업무를 완료하는 데 필요한 시간 예측  
  
- 업무 기간 및 업무 의존도 설정  
  
- 각 업무를 완료하는 데 필요한 역할 결정  
  
  작업 분할 구조의 프로젝트 일정은 대화식 Gantt 차트로 만들어 형태나 사용 면에서 익숙합니다.  
  
## <a name="create-a-work-breakdown-structure-for-a-project"></a>프로젝트에 대한 작업 분할 구조 만들기  
 프로젝트에서 업무 순서를 나타낼 작업 분할 구조를 만듭니다. 작업 분할 구조에는 업무, 각 업부에 대한 요구 사항, 수익 및 비용 정보가 포함됩니다. 작업 분할 구조에 다음을 추가할 수 있습니다.  
  
-   계층 구조에서의 업무 순서  
  
-   시작 전에 완료해야 하는 다른 업무(있을 경우)  
  
-   시작 날짜, 종료 날짜 및 업무 기간  
  
-   업무에 필요한 시간  
  
-   필요한 작업자 기술 및 교육  
  
-   업무에 할당된 작업자  
  
-   예상 수익 및 비용  
  
## <a name="task-types"></a>작업 유형  
작업 분할 구조 생성 시 다음 유형의 업무를 사용합니다.  

| 작업 | Description | 
|---------------------------------------|-----------------------------------------------------------------| 
| **프로젝트의 루트 노드** | 프로젝트에 대한 최상위 요약 업무 다른 모든 프로젝트 업무가 그 아래 생성됩니다. 루트 작업의 이름은 프로젝트 이름입니다. 루트 노드의 업무량, 날짜, 기간은 그 아래 계층의 값을 바탕으로 합니다. 루트 노드를 삭제하거나 루트 노드의 속성을 편집할 수 없습니다. | 
| **요약 또는 컨테이너 업무** | 요약 업무는 그 아래에 하위 작업이 있는 업무입니다. 요약 업무는 작업량 또는 비용이 없습니다. 작업량과 비용은 하위 업무의 롤업입니다. 요약 업무의 이름을 변경할 수 있지만 업무량, 날짜, 기간은 자동으로 계산되므로 바꿀 수 없습니다. 요약 업무를 삭제하면 해당 업무와 하위 업무가 모두 삭제됩니다.|  
| **리프 노드 업무** | 리프 노드 업무는 프로젝트에서 가장 분화된 작업을 나타냅니다. 예상된 작업량, 계획된 리소스 수, 계획된 시작일과 종료일, 기간이 포함됩니다.|

  
## <a name="task-hierarchy"></a>업무 계층 구조  
 업무 계층 구조 생성에는 다음 옵션을 사용할 수 있습니다.  
  
- **업무 추가**   업무 계층 구조에서 선택한 위치에 업무를 추가할 수 있습니다. 위치를 선택하지 않으면 새 업무가 마지막에 나타납니다.  
  
- **업무 들여쓰기**   바로 위에 있는 업무의 하위 업무를 들여씁니다.  
  
- **업무 내어쓰기**   업무를 내어쓰면 더 이상 원래 상위 업무의 하위 업무가 아닙니다.  
  
- **위로 이동 및 아래로 이동**   업무를 상위 업무의 계층 구조에서 아래로 이동합니다. 업무를 위나 아래로 이동해도 작업량, 비용, 날짜, 기간에는 영향을 주지 않습니다.  
  
## <a name="task-attributes"></a>업무 특성  
 업무 이름이 완료되어야 하는 작업을 설명합니다. 다양한 업무 특성을 사용하여 업무에 대한 일정 및 직원 구성 요구 사항을 설명할 수 있습니다.  
  
### <a name="schedule-attributes"></a>일정 특성

 - **작업량(시간)**, **리소스 수**, **시작 날짜**, **종료 날짜**, **기간** 에 값을 할당하여 업무에 대한 일정을 정합니다. 
 - **작업량** 은 업무 완료에 걸리는 시간의 추정치입니다.
 - **리소스 수** 는 프로젝트 관리자가 업무에 투입할 리소스의 추정치로 가능한 최상의 일정을 만드는 데 도움이 됩니다. 
 - **기간**(일별)은 업무 완료에 걸리는 작업일 수를 나타냅니다.  
  
### <a name="staffing-attributes"></a>직원 구성 특성

 - **역할**, **리소스 조직 단위**, **리소스 수** 및 **리소스** 는 업무에 필요한 직원 구성을 설명합니다. 
 - **역할** 은 업무 수행에 필요한 리소스 유형을 설명합니다. 
 - **리소스 조직 단위** 는 해당 업무에 구성할 리소스의 조직 단위를 나타내며, 리소스에 대한 단위 판매 가격을 결정할 때 고려하는 사항이므로 업무의 비용 및 판매 추정치에 영향을 미칩니다. 
 - **리소스** 에는 포괄적 리소스 또는 명명된 리소스가 있습니다.  
  
## <a name="task-dependencies"></a>업무 종속성  
 작업 분할 구조에서 하나 이상의 업무 선행 관계를 만들 수 있습니다. 업무에 대한 선행자 필드에 대한 하나 이상의 값을 설정하여 종속될 업무를 표시할 수 있습니다. 선행 값을 업무에 배정할 때 모든 선행 업무가 완료되어야 후속 업무를 시작할 수 있습니다. 업무에 대한 종속성을 설정하면 모든 선행 업무의 최신 종료 시점으로 업무에 계획된 시작 날짜를 다시 계산하게 됩니다. 선행 업무가 일정에 미치는 영향은 업무에서 정의된 업무 모드로 제한되지 않습니다.  
  
## <a name="task-mode"></a>업무 모드  
 업무 모드는 리프 노드 업무 일정을 결정하는 중요한 요소입니다. 모든 업무에는 자동 일정 모드와 수동 일정 모드의 두 가지 업무 모드가 있습니다.  
  
-   **자동 예약**.   업무 모드를 자동으로 예약으로 설정할 경우, 업무 일정 엔진이 다음 업무 특성에 대한 일정 규칙을 사용하여 업무에 대한 일정을 정합니다.  
  
    -   선행 업무  
  
    -   작업량  
  
    -   리소스 수  
  
    -   시작 및 종료 날짜  
  
-   **내 예약 규칙**.   프로젝트의 일정 시작일에 대한 선행 업무가 없는 리프 노드 업무의 시작 날짜. 리프 노드 업무 기간은 항상 시작 및 종료 날짜 간의 작업일 수로 계산됩니다. 업무 일정이 자동으로 설정되는 경우, 일정 엔진은 아래 규칙을 따릅니다.  
  
    -   업무 시작 및 종료 날짜는 반드시 프로젝트의 일정 관리 일정에 따른 작업일 수여야 함  
  
    -   선행 업무가 있는 경우, 선행 업무의 최종 날짜가 업무 시작 날짜  
  
    -   작업량 = 사용자 수 * 기간 * 프로젝트 일정의 표준 작업일의 시간  
  
-   **일정 관리자**.   경우에 따라, 이러한 규칙을 제외할 수 있습니다. 이 경우 업무에 대한 업무 모드를 수동으로 일정을 수립하도록 설정할 수 있습니다. 그러면 일정 엔진이 다른 일정 특성에 대한 값 계산을 중지합니다. 업무의 선행 업무 설정은 항상 종속 업무의 시작 날짜에 영향을 줍니다.  
  
## <a name="create-a-work-breakdown-structure"></a>작업 분할 구조 만들기  
  
1.  **Project Service > 프로젝트** 로 이동합니다.  
  
2.  작업을 원하는 프로젝트를 클릭합니다.  
  
3.  화면 상단 표시줄에서 프로젝트 이름 옆의 아래로 향하는 화살표를 선택한 다음 작업 분할 구조를 클릭합니다.  
  
4.  업무를 추가하려면 **업무 추가** 를 클릭합니다. 업무에 대한 필드를 입력한 후 **저장** 을 클릭합니다.  
  
5.  작업 분할 구조가 완성될 때까지 업무를 추가합니다. 작업 분할 구조를 생성하는 동안 다음을 수행하여 업무를 구성할 수 있습니다.  
  
    -   업무를 선택하고 **들여쓰기** 를 클릭하여 다른 업무 아래로 이동하거나 내어쓰기를 클릭하여 내어 씁니다.  
  
    -   업무를 선택하고 **위로 이동** 또는 **아래로 이동** 을 클릭하여 목록에서 위나 아래로 이동합니다.  
  
    -   **Gantt 숨기기** 를 클릭하여 Gantt 차트를 숨기고 다시 표시하려면 **Gantt 표시** 를 클릭합니다.  
  
    -   **시간 조정** 의 Gantt 차트에 대한 기간을 선택합니다.  
  
6.  프로젝트의 팀 구성원에게 작업 분할 구조에서 지정한 역할을 추가하려면 **프로젝트 팀 생성** 을 클릭합니다.  
  
7.  변경을 마쳤으면, 화면 우측 하단의 **저장** 을 클릭합니다.
