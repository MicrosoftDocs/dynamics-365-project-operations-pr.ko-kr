# <a name="manage-a-proforma-project-invoice"></a>견적 프로젝트 송장 관리 

_**적용 대상:** 라이트 배포 - 견적 송장 거래_

Dynamics 365 Project Operations에서 견적 송장은 Dynamics 365 Sales의 송장에 대한 확장으로 작성됩니다. 그러나 송장 발행과 관련하여 Sales 및 Project Operations 간의 송장 발행 프로세스에는 많은 차이가 있습니다. 예를 들어, Project Operations의 **송장 목록** 페이지에서 새 송장을 생성할 수 없지만 Sales에서는 가능합니다. 이러한 차이점과 확장은 판매 주문에 대한 일반적인 송장과 다른 프로젝트에 대한 송장 프로세스를 지원하기 위해 마련되었습니다.

> [!IMPORTANT]
> 차이 때문에 Sales 및 Project Operations에서 송장을 서로 바꿔서 사용하지 마십시오.

## <a name="project-based-invoice-lines"></a>프로젝트 기반 송장 라인

Project Operations에는 항상 모든 프로젝트 계약 내용에 대해 하나의 송장 라인이 있습니다. 실제가 없는 경우에도 송장 라인이 생성됩니다. 다음 정보는 견적 송장 라인에서 사용할 수 있습니다.

| 필드 | 위치 | 설명 | 다운스트림 영향 |
| --- | --- | --- | --- |
| **송장 ID** | **일반** 탭 | 송장 ID에 대한 참조입니다. 편집할 수 없도록 잠긴 읽기 전용 필드입니다. | 송장 ID 링크를 사용하여 송장 헤더로 돌아갈 수 있습니다. |
| **이름** | **일반** 탭 | 계약 내용 이름에서 기본적으로 설정된 송장 라인 이름입니다. 이 필드는 사용자가 편집할 수 있습니다. | &nbsp; |

| **소계(\)** | **일반** 탭 | 이 송장 라인의 모든 청구 가능한 송장 라인 세부 사항에 있는 총 금액(**세금 + 금액**)의 합계입니다. 편집할 수 없도록 잠긴 읽기 전용 필드입니다. | 이 필드는 송장 헤더에서 최종 금액을 계산하는 데 사용됩니다. |


## <a name="invoice-line-details"></a>송장 라인 세부 정보

프로젝트 송장의 각 송장 라인에는 송장 라인 상세 내역이 포함됩니다. 이러한 라인 상세 내역은 송장 라인에서 참조하는 계약 내용과 관련된 미청구 판매 실제값 및 중요 시점과 관련됩니다. 이 모든 트랜잭션은 **송장 발부 준비 완료** 로 표시됩니다.

**시간 및 자료 송장** 라인의 경우 송장 라인 상세 내역이 **송장 라인** 페이지에서 **청구 가능**, **청구 불가** 및 **무료** 로 그룹화됩니다. **청구 가능 송장 라인** 세부 사항은 송장 라인 합계에 추가됩니다. **무료** 및 **청구 불가능한 실제** 송장 라인 합계에 합산하지 마십시오.

**고정 가격 송장** 라인의 경우 송장 라인 상세 내역은 관련 계약 내용에 **송장 준비** 로 표시된 이정표에서 생성됩니다. 중요 시점에서 송장 라인 세부 정보가 생성되면 중요 시점의 청구 상태가 **고객 송장 만듦** 으로 업데이트됩니다.

### <a name="create-new-invoice-line-details"></a>새 송장 라인 세부 정보 만들기

시간 및 재료 송장 라인에서 신규 송장 라인 세부 정보를 만들 수 있습니다. 이러한 송장 라인 세부 정보는 실제 항목에 의해 뒷받침되지 않습니다. 시간 및 자료 송장 라인의 송장 라인에서 **새로 만들기** 를 선택하여 계약 내용에 포함된 트랜잭션 분류에 대한 신규 송장 라인 세부 정보를 만듭니다.

## <a name="refresh-invoice-transactions"></a>송장 트랜잭션 새로 고침

송장이 생성된 후 발생한 실제가 있는 경우 이러한 실제를 송장에 포함할 수 있습니다.

1. **청구 백로그 보기** 에서 데이터를 **송장 발부 준비 완료** 로 표시합니다.   
2. 견적 송장 초안을 열고 리본 **작업** 에서 **송장 라인 트랜잭션 새로 고침** 을 클릭합니다.

  이렇게 하면 과거 날짜가 지났고 **송장 발부 준비 완료** 로 표시된 모든 실제에 대한 송장 라인 세부 정보가 생성되지만 송장에 포함되지 않습니다.

## <a name="product-based-invoice-lines"></a>제품 기반 송장 라인

Project Operations에서 프로젝트에 적용되지 않는 제품 또는 프로젝트 기반 송장 라인과 함께 모든 프로젝트에 대한 송장 라인을 생성할 수 있습니다. 이러한 송장 라인은 제품 기반 계약 내용으로 생성되며 송장 발부 준비 완료로 표시된 후 제품 기반 송장 라인으로 추가됩니다.

제품 기반 송장 라인을 추가한 후에는 변경할 수 없습니다. 그러나 형식 상 송장 초안에서는 삭제할 수 있습니다.
