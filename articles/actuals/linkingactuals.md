# <a name="transaction-origins---link-actuals-to-their-source"></a>트랜잭션 출처 - 실제를 출처에 연결

_**적용 대상 :** 리소스/비 재고 기반 시나리오를 위한 Project Operations, Lite 배포 - 견적 송장 처리_

시간 항목, 비용 항목, 자재 사용 로그 및 프로젝트 송장과 같은 실제를 소스에 연결하기 위해 트랜잭션 출처 레코드가 생성됩니다.

다음 예는 Project Operations 프로젝트 라이프사이클에서 시간 항목의 일반적인 처리를 보여줍니다.

> ![Project Operations에서 시간 항목을 처리합니다.](media/basic-guide-17.png)
 
1. 시간 항목을 제출하면 비용과 미청구 매출액에 대한 두 개의 분개장 행이 생성됩니다.
2. 시간 항목을 최종 승인하면 비용과 미청구 매출액에 대한 두 개의 실제값이 생성됩니다.
3. 사용자가 프로젝트 청구서를 만들면 미청구 매출액 실제값의 데이터를 사용하여 청구서 행 처리가 생성됩니다.
4. 청구서가 확인되면 다음 두 개의 새 실제값이 생성됩니다: 미청구 매출액 반전 및 청구 매출액 실제값.

이 처리 워크플로의 각 이벤트는 트랜잭션 원본 엔터티의 레코드 생성을 트리거하여 시간 입력, 분개장 항목, 실제 및 송장 라인 세부 정보에 걸쳐 생성된 이러한 레코드 간의 관계를 추적하는 데 도움이 됩니다.
