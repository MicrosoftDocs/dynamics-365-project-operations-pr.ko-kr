# <a name="extending-time-entries"></a>시간 항목 연장

_**적용 대상 :** 리소스/비 재고 기반 시나리오를 위한 Project Operations, Lite 배포 - 견적 송장 처리_

Dynamics 365 Project Operations는 확장 가능한 시간 입력 사용자 지정 컨트롤이 포함됩니다. 이 컨트롤에는 다음과 같은 기능이 있습니다.

- 1주일 동안 가로로 시간 입력
- 일, 행 또는 주별 합계
- 행 또는 주 복사
- HH:mm 또는 HH.hh를 통한 시간 항목(HH.hh로 자동 변환)
- 할당, 예약 또는 약속에서 가져오기

시간 항목 연장은 다음 두 영역에서 가능합니다.
- [자신의 용도에 맞게 사용자 지정 시간 항목 추가](#add)
- [주간 시간 항목 컨트롤 사용자 지정](#customize)

## <a name="add-custom-time-entries-for-your-own-use"></a><a name="add"></a>자신의 용도에 맞게 사용자 지정 시간 항목 추가

시간 항목은 여러 시나리오에서 사용되는 핵심 엔터티입니다. 2020년 4월 Wave 1에서 TESA 핵심 솔루션이 도입되었습니다. TESA는 **설정** 엔터티와 새로운 **시간 항목 사용자** 보안 역할을 제공합니다. **msdyn_duration** 과 직접적인 관련이 있는 새로운 필드, **msdyn_start** 및 **msdyn_end** 도 포함되었습니다. 새로운 엔터티인 보안 역할 및 필드를 사용하면 여러 제품에서 시간에 대한 보다 통합된 접근 방식을 사용할 수 있습니다.


### <a name="time-source-entity"></a>시간 원본 엔터티
| 필드 | 설명 | 
|-------|------------|
| Name  | 시간 항목을 만들 때 선택 값으로 사용되는 시간 원본 항목의 이름입니다. |
| 기본 시간 원본[시간 원본: isdefault] | 기본적으로 하나의 시간 원본만 기본값으로 표시될 수 있습니다. 이 기능을 사용하면 항목이 지정되지 않은 경우 기본적으로 시간 원본을 사용할 수 있습니다. |
|시간 원본 유형[시간 원본: sourcetype] | 원본 유형은 시간 원본을 앱에 연결할 수 있는 옵션(시간 항목 원본 유형)입니다. Microsoft는 190,000,000보다 큰 값을 예약합니다.|


### <a name="time-entries-and-the-time-source-entity"></a>시간 항목 및 시간 원본 엔터티
각 시간 항목은 시간 원본 레코드에 연결됩니다. 이 레코드는 시간 항목을 처리하는 방법과 애플리케이션을 결정합니다.

시간 항목은 항상 시작, 종료 및 기간이 연결된 하나의 연속된 시간 블록입니다.

논리는 다음 상황에서 시간 항목 레코드를 자동으로 업데이트합니다.

- 다음 세 필드 중 두 개가 제공되면 세 번째 필드가 자동으로 계산됩니다. 

    - **msdyn_start**
    - **msdyn_end**
    - **msdyn_duration**

- 필드, **msdyn_start** 및 **msdyn_end** 는 시간대를 인식합니다.
- **msdyn_date** 및 **msdyn_duration** 만 지정하여 생성된 시간 항목은 자정에 시작됩니다. **msdyn_start** 및 **msdyn_end** 필드는 그에 따라 업데이트됩니다.

#### <a name="time-entry-types"></a>시간 항목 유형

시간 항목 레코드에는 관련 애플리케이션에 대한 제출 흐름의 동작을 정의하는 관련 유형이 있습니다.

|레이블 | 값|
|-----|-----|
|휴식 중   |192,355,000|
|여행 | 192,355,001|
|초과 근무   | 192,354,320|
|근무   | 192,350,000|
|결근    | 192,350,001|
|휴가   | 192,350,002|



## <a name="customize-the-weekly-time-entry-control"></a><a name="customize"></a>주간 시간 항목 컨트롤 사용자 지정
개발자는 다른 엔터티에 추가 필드 및 조회를 추가하고 사용자 지정 비즈니스 규칙을 구현하여 비즈니스 시나리오를 지원할 수 있습니다.

### <a name="add-custom-fields-with-lookups-to-other-entities"></a>다른 엔터티에 조회가 있는 사용자 지정 필드 추가
주간 시간 입력 표에 사용자 지정 필드를 추가하는 세 가지 주요 단계가 있습니다.

1. 빠른 만들기 대화 상자에 사용자 지정 필드를 추가합니다.
2. 사용자 지정 필드를 표시하도록 표를 구성합니다.
3. 행 편집 작업 흐름 또는 셀 편집 작업 흐름에 사용자 지정 필드를 추가합니다.

또한 새 필드에 행 또는 셀 편집 작업 흐름에 필요한 유효성 검사가 있는지 확인하십시오. 이 단계의 일부로 시간 입력 상태에 따라 필드를 잠급니다.

### <a name="add-the-custom-field-to-the-quick-create-dialog-box"></a>빠른 만들기 대화 상자에 사용자 지정 필드 추가
사용자 지정 필드를 **시간 항목 빠른 만들기** 대화 상자에 추가합니다. 그런 다음 시간 항목을 추가할 때 **새로 만들기** 를 선택하여 값을 입력할 수 있습니다.

### <a name="configure-the-grid-to-show-the-custom-field"></a>사용자 지정 필드를 표시하도록 표 구성
주간 시간 입력 표에 사용자 지정 필드를 추가하는 두 가지 방법이 있습니다.

  - 보기 사용자 지정 및 사용자 지정 필드 추가
  - 새 기본 사용자 지정 시간 항목 만들기 


#### <a name="customize-a-view-and-add-a-custom-field"></a>보기 사용자 지정 및 사용자 지정 필드 추가

**내 주간 시간 항목** 보기를 사용자 지정하고 사용자 지정 필드를 추가합니다. 보기에서 해당 속성을 편집하여 표에서 사용자 지정 필드의 위치와 크기를 선택할 수 있습니다.

#### <a name="create-a-new-default-custom-time-entry"></a>새 기본 사용자 지정 시간 항목 만들기

이 보기에는 표에 포함하려는 열 외에 **설명** 및 **외부 주석** 필드가 포함되어야 합니다. 

1. 보기에서 해당 속성을 편집하여 표에서 사용자 지정 필드의 위치, 크기 및 기본 정렬 순서를 선택합니다. 
2. **시간 항목 표** 컨트롤이 되도록 이 보기에 대한 사용자 지정 컨트롤을 구성합니다. 
3. 이 컨트롤을 보기에 추가하고 웹, 휴대폰 및 태블릿용으로 선택합니다. 
4. 주간 시간 항목 표에 대한 매개 변수를 구성합니다. 
5. **시작 날짜** 필드를 **msdyn_date** 로 설정하고 **기간 필드** 를 **msdyn_duration** 으로 설정하고 **상태** 필드를 **msdyn_entrystatus** 로 설정합니다. 
6. 기본 보기의 경우 **읽기 전용 상태 목록** 필드가 **192350002,192350003,192350004** 로 설정됩니다. **행 편집 작업 흐름** 필드가 **msdyn_timeentryrowedit** 으로 설정됩니다. **셀 편집 작업 흐름** 필드가 **msdyn_timeentryedit** 으로 설정됩니다. 
7. 이러한 필드를 사용자 지정하여 읽기 전용 상태를 추가 또는 제거하거나 행 또는 셀 편집에 다른 TBX(작업 기반 환경)를 사용할 수 있습니다. 이러한 필드는 이제 정적 값에 바인딩됩니다.


> [!NOTE] 
> 두 옵션 모두 **프로젝트** 및 **프로젝트 작업** 엔터티에서 일부 즉시 필터링을 제거하므로 엔터티에 대한 모든 조회 보기가 표시됩니다. 즉시 관련 조회 보기만 표시됩니다.

사용자 지정 필드에 적합한 작업 흐름을 결정합니다. 표에 필드를 추가한 경우 시간 항목의 전체 행에 적용되는 필드에 사용되는 행 편집 작업 흐름으로 이동해야 합니다. 사용자 지정 필드에 **끝 시간** 에 대한 사용자 지정 필드와 같은 매일 고유한 값이 있는 경우 셀 편집 작업 흐름으로 이동해야 합니다.

작업 흐름에 사용자 지정 필드를 추가하려면 **필드** 요소를 페이지의 적절한 위치로 끌어온 다음 필드 속성을 설정합니다. **소스** 속성을 **시간 항목** 으로 설정하고 **데이터 필드** 속성을 사용자 지정 필드로 설정합니다. **필드** 속성은 TBX 페이지에서 표시 이름을 지정합니다. **적용** 을 선택하여 변경 사항을 필드에 저장한 다음 **업데이트** 를 선택하여 페이지에 대한 변경 사항을 저장합니다.

대신 새 사용자 지정 TBX 페이지를 사용하려면 새 프로세스를 만듭니다. 범주를 **비즈니스 프로세스 흐름** 으로 설정하고, 엔터티를 **시간 항목** 으로 설정하고, 비즈니스 프로세스 유형을 **작업 흐름으로 프로세스 실행** 으로 설정합니다. **속성** 에서 **페이지 이름** 속성은 페이지의 표시 이름 설정해야 합니다. TBX 페이지에 모든 관련 필드를 추가합니다. 프로세스를 저장하고 활성화합니다. 관련 작업 흐름에 대한 사용자 지정 제어 속성을 프로세스의 **이름** 값으로 업데이트합니다.

### <a name="add-new-option-set-values"></a>새 옵션 집합 값 추가
옵션 집합 값을 즉시 입력 필드에 추가하려면 필드의 편집 페이지를 연 다음 **유형** 에서 옵션 집합 옆에 **편집** 을 선택합니다. 사용자 지정 레이블과 색상이 있는 새 옵션을 추가합니다. 새 시간 입력 상태를 추가하려는 경우 즉시 입력 필드라는 이름이 **항목 상태** 가 아닌 **상태** 입니다.

### <a name="designate-a-new-time-entry-status-as-read-only"></a>새 시간 입력 상태를 읽기 전용으로 지정
새 시간 입력 상태를 읽기 전용으로 지정하려면 새 시간 입력 값을 **읽기 전용 상태 목록** 속성에 추가합니다. 시간 항목 표의 편집 가능한 부분은 새 상태가 있는 행에 대해 잠깁니다.
그런 다음 비즈니스 규칙을 추가하여 **시간 항목 행 편집** 및 **시간 항목 편집** TBX 페이지의 모든 필드를 잠금니다. 페이지의 비즈니스 프로세스 흐름 편집기에서 열고 **비즈니스 규칙** 을 선택하여 이러한 페이지의 비즈니스 규칙에 액세스할 수 있습니다. 기존 비즈니스 규칙의 조건에 새 상태를 추가하거나 새 상태에 대한 새 비즈니스 규칙을 추가할 수 있습니다.

### <a name="add-custom-validation-rules"></a>사용자 지정 유효성 검사 규칙 추가
주간 시간 입력 표 경험에 대해 추가할 수 있는 두 가지 유형의 유효성 검사 규칙이 있습니다.

- 빨리 만들기 대화 상자 및 TBX 페이지에서 작동하는 클라이언트 측 비즈니스 규칙.
- 모든 시간 항목 업데이트에 적용되는 서버 측 플러그인 유효성 검사.

#### <a name="business-rules"></a>비즈니스 규칙
비즈니스 규칙을 사용하여 필드를 잠그고 잠금을 해제하고, 필드에 기본값을 입력하고, 현재 시간 입력 레코드의 정보만 필요한 유효성 검사를 정의합니다. 페이지의 비즈니스 프로세스 흐름 편집기에서 열고 **비즈니스 규칙** 을 선택하여 TBX 페이지의 비즈니스 규칙에 액세스할 수 있습니다. 그런 다음 기존 비즈니스 규칙을 편집하거나 새 비즈니스 규칙을 추가할 수 있습니다. 더 많은 사용자 지정 유효성 검사를 위해 비즈니스 규칙을 사용하여 JavaScript를 실행할 수 있습니다.

#### <a name="plug-in-validations"></a>플러그 인 유효성 검사
단일 시간 항목 레코드에서 사용할 수 있는 것보다 더 많은 컨텍스트가 필요한 유효성 검사 또는 표의 인라인 업데이트에서 실행하려는 유효성 검사에 플러그 인 유효성 검사를 사용합니다. 유효성 검사를 완료하려면 **시간 항목** 엔터티에 사용자 지정 플러그 인을 만듭니다.

### <a name="copying-time-entries"></a>시간 항목 복사
보기 **시간 항목 열 복사** 를 사용하여 시간 입력 중에 복사할 필드 목록을 정의합니다. **날짜** 및 **기간** 은 필수 필드이며 보기에서 제거하면 안됩니다.
